<!-- CI matrix design constraints from research — these are HARD requirements, not suggestions -->

Runner assignments (violating these breaks the build):
- linux/amd64    → os: ubuntu-latest          (native amd64, GTK/WebKit via apt)
- linux/arm64    → os: ubuntu-24.04-arm        (native arm64 runner — cross-compile from amd64 fails for Wails v2)
- windows/amd64  → os: ubuntu-latest          (cross-compile via mingw-w64; no Windows runner needed)
- darwin/amd64   → os: macos-latest           (Apple SDK restriction — cannot cross-compile from Linux)
- darwin/arm64   → os: macos-latest           (Apple SDK restriction — cannot cross-compile from Linux)

Anti-patterns that MUST NOT appear:
- CGO_ENABLED=0 (breaks Wails on all platforms)
- os: ubuntu-latest for darwin/* targets (Apple SDK restriction)
- os: ubuntu-latest + GOARCH=arm64 for linux/arm64 (Wails CGo cross-compile fails, issues #1921/#3719)
- libwebkit2gtk-4.1-dev (may cause ABI mismatch on older runtime Linux; use 4.0-dev)
</interfaces>

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write   # Required for softprops/action-gh-release to create releases

jobs:
  build:
    name: Build ${{ matrix.platform }}
    strategy:
      fail-fast: false   # Don't cancel other jobs if one fails — useful for debugging individual targets
      matrix:
        include:
          # linux/amd64 — native build on ubuntu-latest
          - os: ubuntu-latest
            platform: linux/amd64
            artifact: gamevault-go-linux-amd64
            packages: ""

          # linux/arm64 — native arm64 runner (GitHub public preview Jan 2025)
          # NEVER use ubuntu-latest + aarch64-linux-gnu-gcc: Wails CGo cross-compile fails (issues #1921, #3719)
          - os: ubuntu-24.04-arm
            platform: linux/arm64
            artifact: gamevault-go-linux-arm64
            packages: ""

          # windows/amd64 — cross-compile from Linux using mingw-w64
          - os: ubuntu-latest
            platform: windows/amd64
            artifact: gamevault-go-windows-amd64.exe
            cc: x86_64-w64-mingw32-gcc
            packages: gcc-mingw-w64-x86-64

          # darwin/amd64 — MUST use native macOS runner (Apple SDK restriction)
          - os: macos-latest
            platform: darwin/amd64
            artifact: gamevault-go-darwin-amd64

          # darwin/arm64 — MUST use native macOS runner (Apple SDK restriction)
          # macos-latest is M1 (arm64) hardware as of late 2024; can also target amd64
          - os: macos-latest
            platform: darwin/arm64
            artifact: gamevault-go-darwin-arm64

    runs-on: ${{ matrix.os }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.22'
          cache: true

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install Wails CLI
        run: go install github.com/wailsapp/wails/v2/cmd/wails@v2.9.3

      - name: Install Linux system dependencies
        if: startsWith(matrix.os, 'ubuntu')
        run: |
          sudo apt-get update -q
          # Install webkit2gtk-4.0 explicitly (not 4.1) for broader runtime compatibility
          # Wails v2 targets webkit2gtk-4.0 as its primary supported version
          sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.0-dev ${{ matrix.packages }}

      # macOS: WebKit is bundled with the macOS SDK — no additional packages needed

      - name: Install frontend dependencies
        run: |
          cd frontend
          npm ci

      - name: Build
        env:
          CGO_ENABLED: "1"
          CC: ${{ matrix.cc || '' }}
        run: |
          wails build -platform ${{ matrix.platform }} -o ${{ matrix.artifact }}

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.artifact }}
          path: build/bin/${{ matrix.artifact }}
          if-no-files-found: error

  release:
    name: Create GitHub Release
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        with:
          path: artifacts/

      - name: Create release and upload binaries
        uses: softprops/action-gh-release@v2
        with:
          files: artifacts/**/*
          generate_release_notes: true
```

Key decisions encoded in this YAML:
1. `fail-fast: false` — if linux/arm64 or darwin has issues, the other 4 targets still produce artifacts
2. Pinned Wails version `@v2.9.3` — don't use `@latest` (could pull a breaking change)
3. `npm ci` not `npm install` — reproducible installs in CI
4. `libwebkit2gtk-4.0-dev` explicit — avoids 4.0/4.1 ABI mismatch described in RESEARCH.md pitfall 4
5. `if-no-files-found: error` on upload-artifact — fails loudly if build produced no binary
6. `permissions: contents: write` — required for action-gh-release to create releases
